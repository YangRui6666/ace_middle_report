# GM6020双轴云台使用手册

## 1. 系统综述

本云台系统是基于电控第三轮联调项目开发的双轴云台控制系统，采用Yaw-Pitch构型设计，具备两个自由度。

### 1.1 主要特性

- **双轴控制**：Yaw轴`±60°`，Pitch轴`-15°`~+`45°`运动范围
- **多模式运行**：支持自检、搜索、自瞄工作模式
- **安全保护**：多重保护机制，软件限位防护
- **姿态稳定**：基于`BMI088`陀螺仪的姿态解算和补偿

## 2. 硬件架构

### 2.1 核心组件

| 组件 | 型号规格 | 功能说明 |
|------|----------|----------|
| **主控芯片** | `STM32F405RGT6` | 系统主控制器，`HAL库`开发 |
| **姿态传感器** | `BMI088`陀螺仪 | 六轴IMU，`SPI`通信接口 |
| **云台电机** | `GM6020` × 2 | 无刷直流电机，`CAN`总线控制 |
| **通信接口** | `Micro-USB` | 虚拟串口，与视觉系统通信 |

## 3. 工作流程

> 通讯包：用于通讯的指令
>

### 3.1 开机自检

**开机流程：**

1. 系统上电，进行硬件自检
3. **电机自检**：
   - Yaw轴：在`±60°`范围内转动测试
   - Pitch轴：在`±42°`范围内转动测试
3. **IMU检测**：验证陀螺仪通信和数据有效性
5. 自检完成后进入**等待连接**状态

**等待连接状态：**

- **模式**：此时云台启动**对地**坐标轴的**自稳**

### 3.2 工作模式

#### 3.2.1 稳定模式

- **切入条件**：开机自检完成后的**默认工作状态**，也是搜索模式超时后的**回退状态**，以及锁定保护模式解锁后的**返回状态**
- **工作特性**：Yaw、Pitch 电机回零后，保持**对地稳定**
- **切出条件**：
  - 自动切换
    - **通讯**中断超过 **`60s`** 时，自动进入**`3.2.4 锁定保护模式`**

  - 手动切换
    - 接收到 **`0x83`** 指令，切换至**`3.2.2 搜索模式`**
    - 接收到 **`0x84`** 指令，切换至**`3.2.3 自瞄模式`**
    - 接收到 **`0x87`** 指令，切换至**`3.2.4 锁定保护模式`**

#### 3.2.2 搜索模式

- **切入条件**：接收到 **`0x83`** 指令，或自瞄模式超时后的自动降级
- **工作特性**：云台按预设轨迹**自主**运动（正弦函数）
- **切出条件**：
  - 自动切换
    - 超过**`500ms`**未接收到**通讯包**时，**自动降级**至**`3.2.1 稳定模式`**
  - 手动切换
    - 接收到 **`0x84`** 指令，切换至**`3.2.3 自瞄模式`**

#### 3.2.3 自瞄模式

- **切入条件**：接收到 **`0x84`** 指令
- **工作特性**：根据视觉发送的**相机光心与装甲板的夹角误差**，增量修正云台目标`Yaw-Pitch`
- **通信要求**：~~上位机需以不低于 200Hz 的频率发送控制指令，推荐 250Hz 以上（）~~
- **切出条件**：
  - 自动切换
    - 连续 **`500ms`** 未收到**通讯包**时，**自动降级**至**`3.2.2 搜索模式`**
  - 手动切换
    - 接收到 **`0x83`** 指令，切换至**`3.2.2 搜索模式`**

#### 3.2.4 锁定保护模式

- **切入条件**：
  - 接收到 **`0x87`** 指令
  - 连续 **`60s`** 未收到上位机有效通讯包
- **状态特征**：云台停止输出扭矩，维持锁定保护状态
- **切出条件**：
  - 接收到 **`0x88`** 指令，返回 **`3.2.1 稳定模式`**

#### 3.2.5 失能保护模式

***出现该情况时，请联系电控处理***

- **切入条件**：
  - IMU 传感器数据异常或通信故障
  - 电机反馈信号丢失超过 20ms
  - 电机角度解算值超出安全范围
- **状态特征**：电机**断电失能**，失去扭矩输出，程序终止
- **切出条件**：
  - 无法通过任何指令切出

**建议检查当前硬件机械连接，重启云台**

## 4. 通信协议规范

### 4.1 数据帧格式

| 字段 | 字节数 | 数据类型 | 说明 |
|------|--------|----------|------|
| **帧起始** `SOF` | 2 | `0xAA 0x55` | **固定**起始标志 |
| **数据长度** `LEN` | 1 | uint8_t | **数据区**字节长度（不含识别码、CRC、包尾） |
| **识别码** `CMD` | 1 | uint8_t | 指令类型/来源标识 |
| **数据区** `DATA` | n | 自定义结构体 | 实际负载数据（yaw、pitch等） |
| **CRC校验** `CRC16` | 2 | uint16_t | **从`SOF`到`DATA`结束**的CRC16校验（小端序） |
| **包尾** `EOF` | 2 | `0x5A 0xA5` | **固定**结束标志 |

### 4.2 数据完整性

- **校验范围**：从帧起始0xAA开始到数据区结束
- **校验算法**：CRC16/MODBUS
- **字节序**：CRC16采用小端序存储
- **错误处理**：校验失败的数据包将被丢弃

## **上位机编程指南**

枚举定义

```C
// 云台状态定义
typedef enum {
    GIMBAL_MODE_STABLE = 0,		//稳定模式
    GIMBAL_MODE_SEARCH = 1,		//搜索模式
    GIMBAL_MODE_AUTO_AIM = 2,	//自瞄模式
    GIMBAL_MODE_LOCK_PROTECT = 3,	//锁定模式
    GIMBAL_MODE_DISABLE = 4		//失能模式
} GimbalMode;
// 云台错误码定义
typedef enum {
    ERROR_NONE = 0,
    ERROR_MOTOR_OFFLINE = 1,
    ERROR_ENCODER_FAIL = 2,
    ERROR_LOCK = 9
} ErrorCode;
// 锁定原因定义（用于 0x08 报文的 msg 字段）
typedef enum {
    LOCK_REASON_MANUAL = 1,            // 主动锁定
    LOCK_REASON_BOOT_TIMEOUT = 2,      // 历史保留值，当前协议不主动使用
    LOCK_REASON_COMM_TIMEOUT = 3       // 通讯超时
} LockReason;
```

1. **反馈** 

   **识别码**为`0x01`~`0x09`


   - `0x03` 云台发送反馈报文
   
     `DATA`区如下
   
     ```C
     // cmd_id = 0x03 （数据反馈，数据区共 12 字节）
     typedef struct
     {
         int16_t yaw_target;		// 当前 yaw 绝对角度 ×100（度）
         int16_t pitch_target;	// 当前 pitch 绝对角度 ×100（度）
         int16_t roll_target;	// 当前 roll 绝对角度 ×100（度）
         uint32_t time_stamp;    // 毫秒时间戳
         uint8_t mode; 			// 目前云台工作模式
         uint8_t reserved;  		// (保留位)
     } AimControl_t;
     ```

   - `0x08` 云台锁定反馈报文

     `DATA`区如下

     ```C
     // cmd_id = 0x08
     typedef struct
     {
         uint32_t time_stamp;    // 毫秒时间戳
         uint8_t msg;            // 锁定原因，取值见 LockReason
     } LockFeedback_t;
     ```

2. **控制**

   **识别码**为0x82~0x88


   - 0x82 开启数据传输

     `DATA`区为空

     - 开启数据传输后，下位机才会发送数据

   - **0x83 进入搜索模式**
     其中该命令的`DATA`区如下

     ```C
     // cmd_id = 0x83
     typedef struct
     {
         uint32_t time_stamp;	//	时间戳
     }SearchCtl_t;
     ```
     
   - **0x84 自瞄指令**

     其中该命令的`DATA`区如下

     ```C
     // cmd_id = 0x84 
     typedef struct
     {
         int16_t yaw_target;   // 相机光心到装甲板的 yaw 误差角 ×100（度）
         int16_t pitch_target; // 相机光心到装甲板的 pitch 误差角 ×100（度）
         uint32_t time_stamp;	//	时间戳
         uint8_t reserved[2];  // (保留位)
     } AimControl_t;
     ```

     该命令为**增量式误差输入**，不是绝对云台角度。下位机会基于当前实际姿态计算新的绝对控制目标。

   - **0x87 请求进入锁定保护模式**

     `DATA`区为空

   - **0x88 请求退出锁定保护模式**

     `DATA`区为空，解锁后返回稳定模式
